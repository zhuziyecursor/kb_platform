一概述与目标

当前 kb-doc-processor 使用 Apache Tika 作为 PDF 解析引擎。Tika 对扫描版 PDF、复杂表格、数学公式、多栏布局的解析质量有限，尤其在审计行业文档（财务报表、法规文件等）场景下容易丢失结构信息。

MinerU 是基于深度学习的高精度文档解析引擎，支持 OCR、表格还原、公式识别，并提供完整的 FastAPI 服务接口。本方案采用与 RAGFlow 相同的集成模式——不直接运行 MinerU，而是将 PDF 以 HTTP multipart 方式发送给 MinerU 服务端，读取返回的结构化结果后进行后续分块与索引。

🎯 集成原则 1. 零侵入现有分块/嵌入/索引流程——仅替换 Parser 层
2. 支持 Tika ↔ MinerU 运行时切换，保障降级安全
3. 充分利用 MinerU 的结构化输出（类型、页码、坐标），丰富 chunk 元数据
4. 配置驱动，不硬编码，通过 settings.yaml / 环境变量控制

与 RAGFlow 方案的差异

维度RAGFlow 方案本项目适配方案
调用方式HTTP POST /file_parse相同，HTTP POST /file_parse
文件来源用户上传直接传入从 MinIO 下载后再发送
结果处理转换为 sections 传给分块转换为 ParseResult + PageInfo，接入现有 Pipeline
配置管理Docker 环境变量settings.yaml + 环境变量占位符（现有机制）
降级策略无内置降级MinerU 不可达时自动降级至 TikaParser
页码信息提取坐标标签按 page_idx 填充 PageInfo，保留页码供 chunk 使用

二整体架构设计

2.1 集成后的 Pipeline 架构

┌─────────────────────── kb-doc-processor ──────────────────────────────────────────────┐ │ │ │ Kafka Consumer │ │ │ file-ingest 消息 │ │ ▼ │ │ ┌─────────────┐ │ │ │ Pipeline │ process_message() │ │ │ │ │ │ │ ① PARSING │─────▶ ParserFactory（新增） │ │ │ │ │ │ │ │ │ ┌───────┴────────┐ │ │ │ │ │ │ │ │ │ │ MinerUParser TikaParser │ │ │ │ （新增） （现有） │ │ │ │ │ │ │ │ │ │ └───────┬────────┘ │ │ │ │ │ ParseResult（统一接口，不变） │ │ │ │ ▼ │ │ │ ② CLEANING │─────▶ TextCleaner（不变） │ │ │ ③ CHUNKING │─────▶ FixedLength / Semantic / LLMChunker（不变） │ │ │ ④ EMBEDDING│─────▶ EmbeddingClient（不变） │ │ │ ⑤ SAVING │─────▶ DB + Kafka publish（不变） │ │ └─────────────┘ │ │ │ └────────────────────────────────────────────────────────────────────────────────────────┘ │ MinIO: 下载 PDF bytes ▲ 上传时存入 │ │ ▼ │ ┌──────────────────────────────────┐ ┌──────────────────┐ │ MinerU FastAPI 服务 │ │ MinIO (kb-raw) │ │ POST /file_parse (multipart) │ │ │ │ 返回 ZIP (content_list.json) │ └──────────────────┘ └──────────────────────────────────┘

2.2 MinerU 解析数据流

Step 1 Pipeline → MinioClient.get_object(src_path) → MinIO # 获取 PDF bytes Step 2 Pipeline → ParserFactory.create(config) → MinerUParser # 根据配置选择解析器 Step 3 MinerUParser → check_health() → MinerU /health # 健康探测（可选，每次或定期） Step 4 MinerUParser → POST /file_parse (multipart) → MinerU # 上传 PDF，请求 ZIP 返回 Step 5 MinerU → ZIP (content_list.json + images) → MinerUParser # 解析完成，返回结构化数据 Step 6 MinerUParser → _extract_zip() → tempdir # 解压到临时目录 Step 7 MinerUParser → _read_content_list() → JSON # 读取 content_list.json Step 8 MinerUParser → _to_parse_result() → ParseResult # 转为 ParseResult[PageInfo] Step 9 ParseResult → Pipeline._clean() / _chunk() / ... → downstream # 接入现有流程，无感知变化

2.3 content_list.json → ParseResult 映射规则

MinerU content type取值字段填入 PageInfo.text 方式是否保留
texttext直接拼接✅
listlist_items[]换行符拼接✅
codecode_body带反引号代码块格式✅
equationtext带 $$ 包裹的 LaTeX 文本✅
tabletable_body (HTML) + table_captionHTML 原文（现有 Cleaner 会处理）✅
imageimage_caption[]图片说明文字（无图字节）✅（仅文字）

四核心代码实现

4.1 MinerUParser — HTTP 客户端解析器

新建文件 src/parser/mineru_parser.py：

mineru_parser.py """ MinerU HTTP 客户端解析器 将 PDF bytes 发送给 MinerU FastAPI 服务，解析返回的 ZIP 包， 转换为 ParseResult + PageInfo 接口，无缝接入现有 Pipeline。 """ import io import json import logging import os import tempfile import zipfile from enum import StrEnum from pathlib import Path from typing import Optional import requests from src.config import MinerUConfig, MvpLimits from src.parser import BaseParser, PageInfo, ParseResult logger = logging.getLogger(__name__) class MinerUContentType(StrEnum): """MinerU content_list.json 中 type 字段枚举值""" TEXT = "text" TABLE = "table" IMAGE = "image" EQUATION = "equation" CODE = "code" LIST = "list" DISCARDED = "discarded" class MinerUParser(BaseParser): """ 通过 HTTP 调用 MinerU FastAPI 服务解析 PDF。 与 TikaParser 实现相同的 BaseParser 接口，可互换使用。 """ def __init__(self, config: MinerUConfig, mvp_limits: MvpLimits): self._api_url = config.api_server.rstrip("/") self._backend = config.backend self._server_url = config.server_url # vLLM 后端地址（vlm-http-client 模式使用） self._timeout = config.timeout_seconds self._formula_enable = config.formula_enable self._table_enable = config.table_enable self._parse_method = config.parse_method self._max_file_size_mb = mvp_limits.max_file_size_mb self._max_pages = mvp_limits.max_file_pages # ------------------------------------------------------------------ # # 公开接口：与 TikaParser 一致 # # ------------------------------------------------------------------ # def parse(self, file_bytes: bytes, lang_hints: Optional[list[str]] = None) -> ParseResult: """ 上传 PDF bytes，调用 MinerU /file_parse，返回 ParseResult。 """ max_bytes = self._max_file_size_mb * 1024 * 1024 if len(file_bytes) > max_bytes: raise ValueError( f"File size ({len(file_bytes)} bytes) exceeds limit ({self._max_file_size_mb}MB)" ) # 语言列表：lang_hints 优先，否则默认中文 lang_list = ",".join(lang_hints) if lang_hints else "ch" logger.info( f"[MinerU] Sending PDF to {self._api_url}/file_parse, " f"size={len(file_bytes)/1024:.1f}KB, backend={self._backend}" ) with tempfile.TemporaryDirectory(prefix="mineru_") as tmpdir: zip_path = self._call_api(file_bytes, lang_list, Path(tmpdir)) out_dir = Path(tmpdir) / "extracted" out_dir.mkdir() self._extract_zip(zip_path, out_dir) content_list = self._read_content_list(out_dir) result = self._to_parse_result(content_list) logger.info( f"[MinerU] Parsed successfully: {result.page_count} pages, " f"{sum(len(p.text) for p in result.pages)} chars total" ) return result def check_health(self) -> bool: """探测 MinerU 服务是否可用（/health 端点）""" try: resp = requests.get( f"{self._api_url}/health", timeout=5 ) return resp.status_code == 200 except Exception: return False # ------------------------------------------------------------------ # # 私有方法 # # ------------------------------------------------------------------ # def _call_api(self, file_bytes: bytes, lang_list: str, tmpdir: Path) -> Path: """向 MinerU /file_parse 发送 PDF，将返回的 ZIP 写入 tmpdir，返回路径。""" files = { "files": ("document.pdf", io.BytesIO(file_bytes), "application/pdf") } data = { "output_dir": "./output", "lang_list": lang_list, "backend": self._backend, "parse_method": self._parse_method, "formula_enable": "true" if self._formula_enable else "false", "table_enable": "true" if self._table_enable else "false", "return_md": "true", "return_middle_json": "false", "return_content_list": "true", # 必须为 true，我们依赖此字段 "return_images": "false", # 一期不索引图片二进制 "response_format_zip": "true", # 必须返回 ZIP "start_page_id": "0", "end_page_id": "99999", } # vlm-http-client 后端需要传 server_url if self._server_url: data["server_url"] = self._server_url resp = requests.post( url=f"{self._api_url}/file_parse", files=files, data=data, timeout=self._timeout, stream=True, ) if resp.status_code != 200: raise RuntimeError( f"[MinerU] /file_parse returned HTTP {resp.status_code}: {resp.text[:500]}" ) content_type = resp.headers.get("Content-Type", "") if not content_type.startswith("application/zip"): raise RuntimeError( f"[MinerU] Expected application/zip but got: {content_type}" ) zip_path = tmpdir / "result.zip" with open(zip_path, "wb") as f: for chunk in resp.iter_content(chunk_size=8192): f.write(chunk) logger.debug(f"[MinerU] ZIP saved to {zip_path}, size={zip_path.stat().st_size} bytes") return zip_path def _extract_zip(self, zip_path: Path, out_dir: Path) -> None: """解压 ZIP，去除顶层目录前缀（MinerU 会加一层 output/ 目录）。""" with zipfile.ZipFile(zip_path, "r") as zf: for member in zf.infolist(): # 去掉顶层 output/ 前缀，将文件提取到 out_dir 下 parts = Path(member.filename).parts relative = Path(*parts[1:]) if len(parts) > 1 else Path(member.filename) target = out_dir / relative if member.is_dir(): target.mkdir(parents=True, exist_ok=True) else: target.parent.mkdir(parents=True, exist_ok=True) with zf.open(member) as src, open(target, "wb") as dst: dst.write(src.read()) def _read_content_list(self, out_dir: Path) -> list[dict]: """在解压目录中查找 *_content_list.json 并读取。""" candidates = list(out_dir.rglob("*_content_list.json")) if not candidates: raise FileNotFoundError( f"[MinerU] content_list.json not found in extracted ZIP at {out_dir}" ) content_list_path = candidates[0] with open(content_list_path, "r", encoding="utf-8") as f: data = json.load(f) logger.debug( f"[MinerU] content_list loaded from {content_list_path}: {len(data)} items" ) return data def _to_parse_result(self, content_list: list[dict]) -> ParseResult: """ 将 MinerU content_list 转换为 ParseResult + PageInfo 列表。 按 page_idx 分组，每页构建一个 PageInfo。 """ # 收集每页文本块 page_texts: dict[int, list[str]] = {} for item in content_list: item_type = item.get("type", "") page_idx = item.get("page_idx", 0) text = self._extract_text(item, item_type) if text: page_texts.setdefault(page_idx, []).append(text) if not page_texts: return ParseResult( pages=[PageInfo(page_num=1, text="")], metadata={}, ) # 检查页数限制 max_page_idx = max(page_texts.keys()) total_pages = max_page_idx + 1 if total_pages > self._max_pages: raise ValueError( f"[MinerU] Document has {total_pages} pages, exceeds limit ({self._max_pages})" ) # 构建 PageInfo 列表（保持 0-based page_idx → 1-based page_num 映射） pages = [] for page_idx in sorted(page_texts.keys()): page_text = "\n\n".join(page_texts[page_idx]) pages.append(PageInfo( page_num=page_idx + 1, text=page_text.strip(), )) return ParseResult( pages=pages, metadata={ "parseMethod": "MINERU", "mineruBackend": self._backend, "pageCount": len(pages), }, ) def _extract_text(self, item: dict, item_type: str) -> str: """根据内容类型提取文本，丢弃空白和 discarded 类型。""" match item_type: case MinerUContentType.TEXT: return item.get("text", "").strip() case MinerUContentType.LIST: items = item.get("list_items", []) return "\n".join(str(i) for i in items).strip() case MinerUContentType.CODE: code_body = item.get("code_body", "").strip() return f"```\n{code_body}\n```" if code_body else "" case MinerUContentType.EQUATION: eq = item.get("text", "").strip() return f"$$ {eq} $$" if eq else "" case MinerUContentType.TABLE: body = item.get("table_body", "").strip() captions = item.get("table_caption", []) caption_text = " ".join(captions).strip() parts = [] if caption_text: parts.append(caption_text) if body: parts.append(body) return "\n".join(parts) case MinerUContentType.IMAGE: captions = item.get("image_caption", []) return " ".join(captions).strip() case MinerUContentType.DISCARDED | _: return ""

4.2 ParserFactory — 解析器工厂

新建文件 src/parser/parser_factory.py：

parser_factory.py """ 解析器工厂。 根据 AppConfig 中的 parser 配置段决定使用 TikaParser 还是 MinerUParser。 支持运行时切换，无需修改 Pipeline 代码。 """ import logging from src.config import AppConfig from src.parser import BaseParser from src.parser.tika_parser import TikaParser from src.parser.mineru_parser import MinerUParser logger = logging.getLogger(__name__) SUPPORTED_ENGINES = ("tika", "mineru") def create_parser(config: AppConfig) -> BaseParser: """ 根据 config.parser.engine 创建并返回相应的解析器实例。 engine = "tika" → TikaParser（默认，现有行为） engine = "mineru" → MinerUParser（新增，需配置 mineru.api_server） 当 engine = "mineru" 且服务不可达时，根据 parser.fallback_on_failure 决定是抛出异常还是降级到 TikaParser。 """ engine = config.parser.engine.lower() if engine not in SUPPORTED_ENGINES: raise ValueError( f"Unsupported parser engine '{engine}', must be one of {SUPPORTED_ENGINES}" ) if engine == "tika": logger.info("[ParserFactory] Using TikaParser") return TikaParser(config.tika, config.mvp_limits) # engine == "mineru" if not config.mineru.api_server: msg = "[ParserFactory] engine=mineru but mineru.api_server is not configured" if config.parser.fallback_on_failure: logger.warning(msg + ", falling back to TikaParser") return TikaParser(config.tika, config.mvp_limits) raise ValueError(msg) parser = MinerUParser(config.mineru, config.mvp_limits) # 启动时做一次健康检测 if not parser.check_health(): msg = ( f"[ParserFactory] MinerU service not reachable at {config.mineru.api_server}" ) if config.parser.fallback_on_failure: logger.warning(msg + ", falling back to TikaParser") return TikaParser(config.tika, config.mvp_limits) raise RuntimeError(msg) logger.info( f"[ParserFactory] Using MinerUParser, api_server={config.mineru.api_server}, " f"backend={config.mineru.backend}" ) return parser

六部署方案选型

方案 A — Pipeline 后端 CPU 支持 推荐内网试用

无 GPU 要求，适合内网初期验证。解析精度好于 Tika，速度比 VLM 模式慢。

# 1. 内网服务器安装 MinerU pip install mineru export MINERU_MODEL_SOURCE=modelscope # 2. 启动 API 服务 mineru-api --host 0.0.0.0 --port 8886 # 3. 配置 kb-doc-processor KB_PARSER_ENGINE=mineru KB_MINERU_API_SERVER=http://<host>:8886 KB_MINERU_BACKEND=pipeline

方案 B — vlm-http-client 需要 GPU 服务器

连接已有 Qwen vLLM 实例（http://192.168.30.47:8000），MinerU API 本身轻量，视觉理解能力更强。

# 注意：需先在 GPU 服务器安装 MinerU vLLM 模型 # 或另建专用 vLLM 实例 mineru-vllm-server --port 30000 # MinerU API（不需要 GPU） mineru-api --host 0.0.0.0 --port 8886 # 配置 KB_PARSER_ENGINE=mineru KB_MINERU_API_SERVER=http://<host>:8886 KB_MINERU_BACKEND=vlm-http-client KB_MINERU_SERVER_URL=http://192.168.30.47:30000

八降级策略与监控

8.1 降级触发时机

场景行为配置项
MinerU 服务未启动启动时健康检测失败 → 降级到 TikaKB_PARSER_FALLBACK=true
MinerU 请求超时requests.Timeout 异常 → 整条消息进入失败重试KB_MINERU_TIMEOUT_SECONDS
MinerU 返回非 ZIPRuntimeError → 消息失败，Kafka 可重试-
content_list.json 解析失败FileNotFoundError / JSONDecodeError → 消息失败-
api_server 未配置启动时检测 → 降级到 Tika 或抛出KB_PARSER_FALLBACK
✅ 推荐生产配置 生产环境设置 KB_PARSER_FALLBACK=true，当 MinerU 服务异常时自动回退到 Tika，保证文档处理链路不中断。可通过 Prometheus + Grafana 监控 parseMethod 分布，异常比例升高时告警。
